## 2. Согласованность кода

- **Типы и валидация:** TaskStatus и VALID_STATUSES содержат одни и те же 11 статусов, порядок совпадает.
- **Поток статусов:** Согласован заголовок → (generationPipeline) → Текст готов, ждём картинку → (imageGenerationPipeline после generationTime) → Готово к проверке / Одобрено на публикацию. «На доработку» → тот же generationPipeline (только текст) → Текст готов, ждём картинку → далее картинка по времени.
- **Публикация:** Проверка `task.status === 'Одобрено на публикацию'` в publishing.ts и scheduler — консистентно.
- **Перегенерация картинки:** regenerateImagePipeline по-прежнему ставит «Готово к проверке» (хардкод); не переводит в «Одобрено на публикацию» при выключенной модерации — осознанное упрощение (в плане не требовалось менять regenerateImage).

## 3. Замечания и риски

### 3.1. Неиспользуемая функция

- **writeGenerationResult** в `writer.ts` после разделения пайплайна нигде не вызывается. Текст пишется через **writeTextResult**, картинка — через **writeRegeneratedImage**. Решение: оставить функцию на случай будущего сценария «текст + картинка за один вызов» или удалить как мёртвый код.

### 3.2. Документация вне основного пакета правок

В следующих файлах по-прежнему используются старые формулировки статусов или старый сценарий генерации. На работу кода не влияют, но могут путать при чтении:

| Файл | Что поправить (рекомендация) |
|------|------------------------------|
| `content-factory/ДЕПЛОЙ_VPS.md` | «Время генерации» для статуса «Согласовано» → указать «для статуса „Текст готов, ждём картинку“ (генерация картинки)» |
| `content-factory/README.md` | «ставит „Согласовано“» → «ставит „Согласован заголовок“»; при желании добавить шаг «Текст готов, ждём картинку» |
| `content-factory/ТЕСТИРОВАНИЕ.md` | Все сценарии со «Согласовано»/«Одобрено» → заменить на «Согласован заголовок»/«Одобрено на публикацию»; описать двухшаговый сценарий (сначала текст, потом после 05:00 картинка) |
| `content-factory/АУДИТ_КОДА.md` | Упоминания «Одобрено»/«Согласовано» в чек-листах — обновить под новые названия |
| `content-factory/docs/AUDIT_ТРИ_ИСПРАВЛЕНИЯ.md` | Описание времени генерации и статусов — обновить под новый пайплайн |
| `ПЛАН_РЕАЛИЗАЦИИ.md`, `ТЗ_Контент_Завод_v3.md`, `ТЗ_Контент_Завод_v4.md` | Исторические ТЗ/план; по желанию заказчика обновить формулировки статусов или оставить как архив |

### 3.3. ПРОМПТ_для_создания_Google_таблицы.md

- В списке статусов значение «Текст готов, ждём картинку» содержит запятую; при разборе списка по запятым его можно ошибочно разбить на два. Рекомендация: дать статус в кавычках в пояснении, например: «в т.ч. „Текст готов, ждём картинку“».

### 3.4. Таблицы, созданные до правок

- В существующих Google Таблицах колонка P может отсутствовать. Запись в P (например, `'Задания'!P2`) при первом обращении создаёт колонку — поведение Google Sheets нормальное.
- Выпадающий список в колонке E нужно обновить (перезапуск Apps Script с новым списком статусов или ручное обновление Data Validation). Старые ячейки со «Согласовано»/«Одобрено» не будут распознаны кодом (VALID_STATUSES), строки выпадут из выборки до ручной замены статусов в таблице.

### 3.5. Пост-обработка первой строки

- `cleanArticleFirstLine` обрабатывает только первую строку: снимает префикс `Заголовок:` / `{Заголовок}:` и обрамляющие кавычки. Варианты вроде «Заголовок статьи: Текст» обрабатываются корректно; экзотические форматы (например, заголовок на второй строке) не нормализуются — при необходимости можно расширить логику.
