## 2. text.ts — truncateAtSentence

### Реализация

```ts
export function truncateAtSentence(text: string, limit = 4000): string {
  if (text.length <= limit) return text;
  const cut = text.lastIndexOf('. ', limit - 1);
  if (cut > limit * 0.5) return text.slice(0, cut + 1).trimEnd();
  return text.slice(0, limit);
}
```

- Ищет последнюю подстроку `. ` (точка + пробел) в диапазоне `[0, limit - 1]`.
- Если такая позиция есть и она больше половины лимита (`cut > limit * 0.5`) — обрезка по границе предложения, иначе жёсткий срез на `limit`.
- Условие `cut > limit * 0.5` предотвращает обрезку в начале текста (если первая точка далеко от лимита — лучше жёсткий срез, чем отрезать почти весь текст).

### Проверки

- **Текст ≤ limit:** возвращается без изменений — ок.
- **Текст > limit, есть `. ` до limit:** обрезка по последней точке, конец предложения сохранён — ок.
- **Текст > limit, точки нет (один длинный абзац):** `lastIndexOf` даёт -1, `cut > limit * 0.5` ложно — жёсткий срез на limit — приемлемый fallback.
- **Граница предложения в русском:** используется `. ` (точка + пробел). Многоточие, сокращения (например «т.д.») могут дать лишнюю точку — в практике постов это редкость; при необходимости можно расширить поиск (например, `\n` как вторичная граница) — не обязательно на текущем этапе.
- **Пустая строка, limit 0:** `text.length <= limit` → return text; при limit 0 и непустом text — cut будет -1 или < 0, срез на 0 — ок.

### Рекомендации

- Оставить как есть. При появлении кейсов с обрезкой внутри длинного абзаца без точек — рассмотреть учёт `\n` в качестве альтернативной границы.

## 3. draft.ts — заголовок, лимит 4000, max_tokens, обрезка

### Реализация

- **Заголовок:** в userContent добавлен блок: «ВАЖНО: первой строкой статьи выведи БУКВАЛЬНО, без каких-либо изменений этот заголовок: "${headline}"». Заголовок повторён явно в кавычках; инструкция «Не перефразируй, не сокращай, не добавляй слова».
- **Длина:** «Напиши SEO-статью СТРОГО до 4000 символов», «Статья должна быть ЗАВЕРШЁННОЙ», «когда приближаешься к 4000 символам — заканчивай мысль и пиши призыв к действию».
- **max_tokens:** 2048 → 1500.
- **Итог:** `truncateAtSentence(res.choices[0]?.message?.content ?? '', 4000)` вместо `slice(0, 4000)`.
- **Дефолт system:** «До 4000» заменён на «СТРОГО до 4000 символов».

### Проверки

- **headline с кавычками или переносами:** в шаблоне используется `"${headline}"` — если в headline есть кавычки, они попадут в промпт; модель должна копировать строку как есть. Критичных инъекций в логику нет.
- **Пустой content от API:** `?? ''` даёт пустую строку, truncateAtSentence('', 4000) вернёт '' — падения нет.
- **1500 токенов:** ~4000 символов русского текста ≈ 1100–1400 токенов; 1500 даёт небольшой запас и ограничивает верхнюю границу вывода — соответствует задумке.

### Рекомендации

- Нет. Реализация соответствует плану.

## 4. humanize.ts — первая строка, лимит 4000, max_tokens, обрезка

### Реализация

- **Дефолт systemContent:** добавлены фразы: «Итоговый текст СТРОГО до 4000 символов, ЗАВЕРШЁННЫЙ», «Сохрани структуру и длину черновика», «Не обрезай концовку с призывом к действию», «Не изменяй первую строку текста — это заголовок статьи, он должен остаться точно таким же».
- **max_tokens:** 2048 → 1500.
- **Итог:** `truncateAtSentence(res.choices[0]?.message?.content ?? '', 4000)` вместо `slice(0, 4000)`.

### Проверки

- **Промпт 3 задан в таблице:** используется `prompt3?.trim()`; если заказчик переопределил Промпт 3, инструкции про первую строку и 4000 символов будут только в дефолте. То есть при кастомном Промпте 3 защита «не меняй первую строку» и «до 4000» может отсутствовать — зависит от формулировок в листе. Приёмлемо: кастомный промпт под полный контроль заказчика.
- **Черновик уже с неправильным заголовком:** humanize просит не менять первую строку; если в draft уже попал перефразированный заголовок, humanize его сохранит. Исправление — на этапе draft (уже усилен).
- **Пустой content:** аналогично draft — ок.

### Рекомендации

- В РУКОВОДСТВО_ПОЛЬЗОВАТЕЛЯ при желании добавить примечание: при использовании своего Промпта 3 полезно сохранять формулировки про «не менять первую строку» и «до 4000 символов», если нужна та же политика. Не блокер.

## 5. generation.ts — финальная обрезка

### Реализация

- Импорт `truncateAtSentence` из `../utils/text`.
- `previewText = truncateAtSentence(finalText, 4000)` вместо `finalText.slice(0, 4000)`.
- Лог «Text exceeded 4000 chars, truncating» сохранён — срабатывает при len > 4000 перед обрезкой; после truncateAtSentence длина может быть ≤ 4000 (по границе предложения).

### Проверки

- **finalText уже ≤ 4000:** truncateAtSentence возвращает текст без изменений — ок.
- **finalText > 4000:** обрезка по предложению или жёсткая; в таблицу и в пост уходит завершённый фрагмент — ок.
- **Запись в ячейку:** writer.ts использует `preview.slice(0, MAX_CELL_PREVIEW)` (32000); наша обрезка 4000 укладывается — конфликта нет.

### Рекомендации

- Нет. Цепочка согласована.

## 7. Итог

- **Заголовок:** инструкция и дублирование текста заголовка в промпте устраняют подмену своим вариантом; humanize дополнительно фиксирует первую строку в дефолтном Промпте 3.
- **Длина:** ограничение max_tokens 1500 не даёт модели выдать избыточный объём; промпт явно требует завершённость и контроль длины; truncateAtSentence гарантирует, что при любом превышении обрезка идёт по границе предложения, а не на полуслове.
- **Риски:** при кастомном Промпте 3 без указания про первую строку и 4000 — поведение зависит от формулировок заказчика; при необходимости можно описать это в руководстве.
- **Обратная совместимость:** лимит остался 4000 (Telegram до 4096); поведение стало строже и предсказуемее. Изменения готовы к продакшену.
