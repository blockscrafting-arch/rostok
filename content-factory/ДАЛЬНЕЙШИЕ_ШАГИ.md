# Дальнейшие шаги: Git, CI, VPS

## 1. Git — залить проект

```bash
cd content-factory
git init
git add .
git status   # убедись, что .env и secrets/ не в списке (они в .gitignore)
git commit -m "Контент-Завод: таблица, пайплайны, Telegram, S3"
```

Создай репозиторий на GitHub/GitLab (без README), затем:

```bash
git remote add origin https://github.com/ТВОЙ_ЛОГИН/content-factory.git
git branch -M main
git push -u origin main
```

**Важно:** в репозитории не должно быть `.env`, `secrets/`, `node_modules/` — они уже в `.gitignore`.

---

## 2. GitHub Actions (CI)

Можно включить простой пайплайн: на каждый push проверять сборку.

- Файл уже создан: `.github/workflows/ci.yml`
- При пуше в `main` будет запускаться `npm ci` и `npm run build`.
- Секреты (ключи, токены) в CI не подставляем — только проверка, что код собирается.

После первого пуша зайди в репозиторий → вкладка **Actions** — там будут запуски.

## 3. VPS (Beget) — деплой

По плану приложение крутится на VPS заказчика (Beget), под PM2.

### 3.1. Однократная настройка VPS

- Подключись по SSH к VPS.
- Установи Node.js 20 LTS, PM2, Git (если ещё нет):

```bash
# Пример для Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs git
sudo npm install -g pm2
```

### 3.2. Клонирование и первый запуск

```bash
cd /home/ваш_пользователь   # или куда разрешён доступ
git clone https://github.com/ТВОЙ_ЛОГИН/content-factory.git
cd content-factory
npm ci
```

### 3.3. Переменные окружения на VPS

Создай на сервере файл `.env` (скопируй с локального, не коммить):

```bash
nano .env
# вставь все переменные: GOOGLE_SERVICE_ACCOUNT_KEY, SPREADSHEET_ID, OPENROUTER_*, TELEGRAM_*, S3_*, YANDEX_* и т.д.
```

Файл ключа Google тоже должен быть на VPS:

```bash
mkdir -p secrets
# скопируй google-sa-key.json в secrets/ (scp с локальной машины или вставь содержимое)
```

Путь в `.env` должен совпадать с расположением ключа, например:

```env
GOOGLE_SERVICE_ACCOUNT_KEY=./secrets/google-sa-key.json
```

### 3.4. Сборка и запуск под PM2

```bash
npm run build
pm2 start ecosystem.config.js
pm2 save
pm2 startup   # выполни команду, которую выведет pm2 (для автозапуска после перезагрузки)
```

Проверка:

```bash
pm2 status
pm2 logs content-factory
```

## 4. Дальнейшие обновления (деплой с Git)

После правок в коде:

```bash
git add .
git commit -m "описание изменений"
git push
```

На VPS:

```bash
cd /path/to/content-factory
git pull
npm ci
npm run build
pm2 restart content-factory
```

При необходимости можно позже добавить деплой через GitHub Actions (SSH на VPS + те же команды), но для начала ручного `git pull` + `pm2 restart` достаточно.
