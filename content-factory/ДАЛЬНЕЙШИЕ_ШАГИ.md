# Дальнейшие шаги: Git, CI, VPS

Актуальное состояние проекта и куда смотреть.

---

## 1. Репозиторий и CI

- **Репозиторий:** [blockscrafting-arch/rostok](https://github.com/blockscrafting-arch/rostok) (в нём папка `content-factory`).
- **CI:** `.github/workflows/ci.yml` в **корне** репозитория. На каждый push в `main`/`master`:
  - `npm ci` в `content-factory`
  - Typecheck: `tsc --noEmit`
  - Build: `npm run build` (esbuild)
  - Tests: `npm run test:run` (vitest)
- Секреты в CI не подставляются — только проверка сборки и тестов.

---

## 2. Деплой на VPS

Подробная инструкция: **[ДЕПЛОЙ_VPS.md](./ДЕПЛОЙ_VPS.md)**.

Кратко:
- **Вариант A (Docker):** установить Docker, клонировать репо, создать `.env` и `keys/service-account.json`, затем `docker compose up -d`.
- **Вариант B (Node на хосте):** Node 20 → клонировать → `.env` и `keys/service-account.json` → `npm ci` → `npm run build` → запуск через **systemd** или **PM2** (команды и пути в ДЕПЛОЙ_VPS.md).

Ключ Google: путь в `.env` — `GOOGLE_SERVICE_ACCOUNT_KEY=keys/service-account.json`; файл кладётся в `content-factory/keys/service-account.json`.

---

## 3. Обновление на VPS после правок в коде

```bash
cd ~/rostok/content-factory   # или твой путь
git pull
npm ci
npm run build
pm2 restart content-factory
# или при systemd: sudo systemctl restart content-factory
```

---

## 4. Полезные команды

| Задача | Команда |
|--------|--------|
| Логи (PM2) | `pm2 logs content-factory` |
| Статус (PM2) | `pm2 status` |
| Логи (systemd) | `journalctl -u content-factory -f` |
| Перезапуск (PM2) | `pm2 restart content-factory` |
| Перезапуск (systemd) | `sudo systemctl restart content-factory` |
