# Руководство пользователя системы «Контент-Завод»

**Рабочая таблица:** [Контент-Завод — Google Таблицы](https://docs.google.com/spreadsheets/d/1yM0-7hHT3Cf2JzzN6PLGSQykZ7mzK1Wzz4ZtF91qHaQ/edit?gid=636828865#gid=636828865)

Система «Контент-Завод» — автоматизированный конвейер для создания SEO-оптимизированных статей для блога/канала питомника. Управление идёт через одну Google Таблицу, система работает на сервере (VPS). Готова к тестам и работе.

---

## 1. Настройка системы

### 1.1. Лист «Настройки» в таблице

На листе «Настройки» укажите:

| Параметр | Что вписать | Описание |
|----------|-------------|----------|
| **Роль** | Текст | Роль ИИ (например: *Ведущий агроном питомника...*) |
| **Промпт 1** | Текст | Инструкция для генерации заголовков и подбора 5–10 КЗ к каждому. Плейсхолдеры: `{keyword}`, `{keywords}`, `{count}` (число заголовков из «Макс. заголовков») |
| **Промпт 2** | Текст | Инструкция для чернового SEO-текста |
| **Промпт 3** | Текст | Инструкция для стилизации текста («очеловечивание») |
| **Промпт картинки** | Текст | Инструкция для генерации изображения без референса. Плейсхолдер: `{headline}`. Пусто — используется встроенный промпт. |
| **Промпт картинки с референсом** | Текст | То же, когда есть референсное фото из Справочника фото. Плейсхолдер: `{headline}`. Пусто — встроенный промпт. |
| **ДНК Бренда** | (ссылки в таблице, доступ настроен) | Формат содержимого — п. 1.2 |
| **Справочник каталога** | (ссылки в таблице, доступ настроен) | Формат содержимого — п. 1.3 |
| **Справочник фото** | (ссылки в таблице, доступ настроен) | Формат содержимого — п. 1.4 |
| **Шаблон UTM** | Текст | Например: `?utm_source=dzen&utm_campaign={campaign}` |
| **Модель граундинга** | Идентификатор модели OpenRouter | Поиск фактов (например, `perplexity/sonar`). Пусто — значение из переменных окружения на сервере. |
| **Модель текста** | Идентификатор модели OpenRouter | Черновик и очеловечивание (например, `deepseek/deepseek-chat`). Пусто — из env. |
| **Модель картинки** | Идентификатор модели OpenRouter | Генерация изображений (например, `google/gemini-3.1-flash-image-preview`). Пусто — из env. |
| **Макс. заголовков** | Число | Сколько заголовков генерировать на одно ключевое слово (по умолчанию: 30). Меньше — дешевле и быстрее. |
| **Макс. статей в день** | Число | Лимит публикаций в сутки (например, `5`) |
| **Режим модерации** | `вкл` или `выкл` | «вкл» — статус «Готово к проверке», «выкл» — сразу «Одобрено» |
| **Время сводки** | Время | Когда отправлять дневной отчёт в Telegram (например, `21:00`) |

Канал для публикации задаётся только в переменных окружения на сервере (TELEGRAM_CHANNEL_ID), в таблице его указывать не нужно.

#### Переменные для промпта картинки

В полях **Промпт картинки** и **Промпт картинки с референсом** можно использовать плейсхолдер:

- **`{headline}`** — подставляется заголовок статьи (колонка «Заголовок» на листе «Задания»).

Референсное фото в текст промпта не подставляется: оно выбирается автоматически по заголовку из Справочника фото и передаётся в модель отдельно (как изображение).

### 1.2. Документ «ДНК Бренда»

**Назначение:** весь текст из документа подставляется в промпт стилизации (Промпт 3). ИИ переписывает черновик в стиле бренда по этому тексту.

**Как заполнить документ:** обычный текст — описание тона, целевой аудитории, правил стиля, запретов и т.п. Форматирование и абзацы сохраняются. Ограничений по формату нет.

### 1.3. Документ «Справочник каталога»

**Назначение:** по заголовку статьи система подбирает раздел каталога и подставляет нужный URL в UTM-ссылку.

**Как заполнить документ:** в каждой строке — одна пара: **название раздела** и **URL страницы**, разделённые табуляцией (Tab) или вертикальной чертой с пробелами (|). Одна строка = один раздел.

Пример:
```
Розы	https://site.ru/catalog/roses
Гортензии	https://site.ru/catalog/hydrangea
Пионы | https://site.ru/catalog/peonies
```

Если в заголовке статьи встречается слово «Розы», в UTM попадёт ссылка на каталог роз. Если совпадения нет — берётся первый URL из списка.

### 1.4. Документ «Справочник фото»

**Назначение:** по заголовку статьи система выбирает референсное фото сорта/раздела и генерирует картинку на его основе.

**Как заполнить документ:** в каждой строке — **название сорта или раздела** и **прямая ссылка на картинку** (URL изображения), разделённые табуляцией (Tab) или вертикальной чертой (|). Одна строка = один сорт/раздел.

Пример:
```
Розы	https://example.com/photos/rose.jpg
Гортензия	https://example.com/photos/hydrangea.jpg
Пион | https://example.com/photos/peony.jpg
```

Ссылка должна вести именно на файл изображения (jpg, png и т.д.), а не на страницу. Если в заголовке есть «Розы», будет использовано фото роз. Если совпадения нет — берётся первое фото из списка.

## 2. Как создавать статьи

Вся работа идёт на листе **«Задания»**.

### Шаг 1: Сбор заголовков

1. В колонку **Ключевое слово** (A) впишите тему (например, `Роза Глория Дей`).
2. В **Лимит частотности** (B) укажите лимит запросов (например, `300` или `300-500`).
3. Поставьте статус **Новое** (E).

Система запросит данные в Яндекс.Wordstat, передаст их ИИ, который сгенерирует 30 заголовков и подберёт к каждому 5–10 релевантных НЧ-запросов. Создаётся **30 отдельных строк** — каждая с одним заголовком в C и своим набором КЗ в D. Это 30 разных статей по одной теме.

### Шаг 2: Выбор заголовков

1. Просмотрите все 30 строк с заголовками.
2. **Удалите** строки с заголовками, которые не нужны.
3. В оставшихся строках поставьте статус **Согласовано** — по одной или сразу по всем нужным.

Система для каждой строки со статусом **Согласовано** выполнит граундинг, напишет черновик, перепишет его по ДНК бренда, сгенерирует картинку (с референсом из Справочника фото, если есть совпадение), посчитает UTM по Справочнику каталога, заполнит Превью, Картинку, Стоимость и поставит статус **Готово к проверке** (или **Одобрено**, если модерация выключена).

### Шаг 3: Проверка и доработка

- **Всё устраивает:** поставьте статус **Одобрено**.
- **Нужно переписать текст:** впишите правки в **Комментарий** (O), поставьте статус **На доработку**. Система перепишет текст с учётом комментария.
- **Нужна другая картинка:** поставьте статус **Перегенерировать картинку**. Система перегенерирует только изображение и пересчитает его стоимость.

### Шаг 4: Публикация

При статусе **Одобрено** система в рамках лимита «Макс. статей в день» отправляет **только текст** (с жирным шрифтом) в Telegram-канал, записывает ссылку на пост в таблицу и ставит статус **Опубликовано**.

## 3. Статусы

Используйте точное написание (лучше выбирать из выпадающего списка):

- **Новое** — задача создана, ждётся сбор семантики.
- **На согласовании** — сгенерированы 30 заголовков (30 строк), нужно выбрать нужные и поставить «Согласовано».
- **Согласовано** — заголовок выбран, запускается генерация текста и картинки.
- **Готово к проверке** — текст и картинка готовы, ждётся решение.
- **На доработку** — переписать текст с учётом комментария.
- **Перегенерировать картинку** — перерисовать только картинку.
- **Одобрено** — утверждено, будет опубликовано в канал.
- **Опубликовано** — статья уже в канале.
- **Ошибка** — сбой (смотреть лист «Лог» или логи сервера).

---

## 4. Сводка и статистика

- **Ежедневная сводка:** в заданное в настройках время (например, 21:00) в Telegram приходит отчёт за день/неделю/месяц и средняя стоимость статьи в $.
- **Лист «Статистика»:** автоматический учёт затрат (токены, стоимость текста и картинок). Вручную ничего не заполнять.
- **Лист «Лог»:** сюда пишутся только ошибки. Если статус сменился на «Ошибка», причину смотреть здесь.
