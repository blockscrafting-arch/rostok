# Как проверить каждую функцию и смотреть логи

## 1. Запуск с логами в консоль

**Локально (удобно для отладки):**
```bash
cd content-factory
npm run dev
```
Все логи (INFO, WARN, ERROR) пойдут в этот терминал. Оставь его открытым.

**На VPS (приложение уже крутится под PM2):**
```bash
pm2 logs content-factory
```
Логи в реальном времени. Второй терминал/вкладка — для команд ниже.

---

## 2. Какая строка таблицы → какой пайплайн

| Статус в таблице | Что запускается | Что смотреть в логах |
|------------------|-----------------|----------------------|
| **Новое** | Semantics: Wordstat → заголовки → запись в таблицу | `Semantics done` или `Semantics pipeline error` |
| **Согласовано** | Generation: граундинг → черновик → картинка → запись | `Generation done` или `Generation pipeline error` |
| **На доработку** | Generation (режим доработки по комментарию) | `Revision pipeline error` при ошибке |
| **Одобрено** | Publishing: публикация в Telegram-канал + уведомление | `Published` или `Publishing pipeline error` |
| Время сводки (настройка «Время сводки») | Ежедневная сводка в Telegram | `Daily summary error` при ошибке |

Один цикл планировщика обрабатывает все подходящие строки, затем ждёт `POLL_INTERVAL_MS` (по умолчанию 60 с) и повторяет.

## 3. Пошаговая проверка функций

Перед шагами: **запусти логи** (`npm run dev` или `pm2 logs content-factory`) и открой таблицу.

### Semantics (Wordstat + заголовки)
1. В листе «Задания» добавь строку: ключевое слово, статус **Новое**.
2. Подожди один цикл (до 1–2 минут).
3. В логах: `Semantics done` с `keyword` и `headlinesCount`, либо `Semantics pipeline error` с текстом ошибки.
4. В таблице: появится **30 строк** (исходная + 29 новых) — каждая с одним заголовком, НЧ-запросами и статусом «На согласовании».

### Generation (генерация текста и картинки)
1. Удали ненужные строки с заголовками. В оставшихся поставь статус **Согласовано** (по одной или сразу по нескольким).
2. Подожди цикл. Генерация дольше (30–60 с и больше).
3. В логах: `Generation done` с `headline` и `costTotalRub`, либо `Generation pipeline error`.
4. В таблице: «Готово к проверке» или «Одобрено» (если модерация выключена), заполнятся превью, источники, картинка, стоимость.

### Publishing (пост в канал + уведомление)
1. Поставь статус **Одобрено** у задания с заполненным превью.
2. Подожди цикл.
3. В логах: `Published` с `postUrl` и `headline`, либо `Publishing pipeline error`.
4. В Telegram: сообщение в канал и уведомление «Опубликовано: …» в чат(ы) из `TELEGRAM_NOTIFY_CHAT_ID`.

### Уведомления об ошибках
- Сделай так, чтобы один из шагов упал (например, неверный API-ключ или отключи сеть).
- В логах: соответствующая строка `... pipeline error` и запись в лист «Лог».
- В Telegram придёт сообщение вида «❌ …: все 3 попытки провалились» (если падает в withRetry).

### Ежедневная сводка
- В настройках укажи «Время сводки» на ближайшие 2–3 минуты (например, текущее время + 2 минуты).
- Дождись этого времени в цикле.
- В логах не должно быть `Daily summary error`; в Telegram — сообщение «Сводка за день» с числом статей и расходом.

---

## 4. Что смотреть в логах

- **`[timestamp] INFO`** — успешные шаги (`Semantics done`, `Generation done`, `Published`, `Content-Factory started`).
- **`[timestamp] WARN`** — предупреждения (пустой ДНК, справочник каталога, справочник фото, Wordstat report failed).
- **`[timestamp] ERROR`** — ошибки с `message` и `stack`; дублируются в лист «Лог».
- При ошибке пайплайна в логах будет и тип задачи (keyword/headline), и полный текст ошибки — по ним можно понять, что сломалось.

---

## 5. Коротко

1. Запусти логи: **`npm run dev`** (локально) или **`pm2 logs content-factory`** (VPS).
2. Меняй в таблице статусы: **Новое** → Semantics, **Согласовано** → Generation, **Одобрено** → Publishing.
3. Смотри в логах соответствующие `... done` или `... pipeline error` и при необходимости лист «Лог» в таблице.
