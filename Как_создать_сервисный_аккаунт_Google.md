# Как создать сервисный аккаунт Google (для Контент-Завод)

Сервисный аккаунт нужен, чтобы программа на сервере могла читать и писать в Google Таблицу и в Google Документы (ДНК бренда, справочник каталога) без входа в браузере.

---

## 1. Открыть Google Cloud Console

Перейди по ссылке: **[console.cloud.google.com](https://console.cloud.google.com/)**

Войди в тот же аккаунт Google, в котором создана таблица «Контент-Завод».

---

## 2. Создать проект (если ещё нет)

- Вверху страницы нажми на название проекта рядом с логотипом Google Cloud.
- Нажми **«Создать проект»** (или **New Project**).
- **Название проекта:** например `Контент-Завод` или `Content Factory`.
- **Организация:** оставь «Без организации» (No organization), если не используешь Google Workspace.
- Нажми **«Создать»** и дождись создания. Выбери этот проект в переключателе проектов.

---

## 3. Включить нужные API

- В меню слева: **«API и сервисы»** → **«Библиотека»** (или **APIs & Services** → **Library**).
- Найди и открой **«Google Sheets API»** → нажми **«Включить»** (Enable).
- Вернись в «Библиотеку», найди **«Google Docs API»** → нажми **«Включить»** (нужно для чтения ДНК бренда и справочника каталога из ссылок в настройках).

---

## 4. Создать сервисный аккаунт

- В меню слева: **«IAM и администрирование»** → **«Сервисные аккаунты»** (или **IAM & Admin** → **Service Accounts**).
- Нажми **«+ Создать сервисный аккаунт»** (Create Service Account).

Заполни:

| Поле | Что ввести |
|------|------------|
| **Имя сервисного аккаунта** | Например: `content-factory-sheets` или `Контент-Завод` |
| **Идентификатор** | Подставится сам (можно не менять) |
| **Описание** | По желанию: «Доступ к таблице и документам Контент-Завод» |

Нажми **«Создать и продолжить»** (Create and Continue).

- **Шаг «Предоставить доступ»:** можно ничего не выбирать — доступ к таблице дашь потом через «Поделиться» в самой таблице. Нажми **«Продолжить»**.
- **Шаг «Предоставить доступ пользователям»:** пропусти, нажми **«Готово»** (Done).

---

## 5. Создать и скачать JSON-ключ

- В списке сервисных аккаунтов найди только что созданный (например, `content-factory-sheets`).
- Нажми на него (по строке).
- Перейди на вкладку **«Ключи»** (Keys).
- Нажми **«Добавить ключ»** → **«Создать ключ»** (Add key → Create new key).
- Выбери тип **JSON**.
- Нажми **«Создать»** — файл автоматически скачается.

Сохрани файл в безопасное место. Для проекта Контент-Завод его обычно кладут в папку `content-factory/secrets/` и называют, например, `google-sa-key.json`.

**Важно:** этот файл содержит секретный ключ. Не загружай его в git, не отправляй в чаты и не свети в публичных репозиториях. В `.gitignore` уже есть `secrets/` и `*.json` в секретах.

---

## 6. Узнать email сервисного аккаунта

- На странице сервисного аккаунта вверху указан **«Электронный адрес»** (Client email). Он выглядит так:  
  `content-factory-sheets@ваш-проект.iam.gserviceaccount.com`

Скопируй этот email — он понадобится на следующем шаге.

---

## 7. Дать доступ к таблице и документам

### Таблица «Контент-Завод»

- Открой свою Google Таблицу (ту, что создал по промпту).
- Нажми **«Настройки доступа»** (Share) в правом верхнем углу.
- В поле «Добавить пользователей» вставь **email сервисного аккаунта** (из шага 6).
- Роль: **«Редактор»** (Editor), чтобы программа могла читать и записывать листы.
- Сними галку «Уведомить пользователей», если не хочешь отправлять письмо (сервисный аккаунт почту не читает).
- Нажми **«Готово»** / **«Поделиться»**.

### Документы (ДНК бренда, справочник каталога)

Если в настройках таблицы указаны ссылки на Google Документы, каждому такому документу тоже нужно выдать доступ:

- Открой документ по ссылке.
- **«Настройки доступа»** → добавь тот же **email сервисного аккаунта** с ролью **«Читатель»** (Viewer).
- Сохрани.

---

## 8. Подставить путь к ключу в проект

В файле `.env` в папке `content-factory` укажи путь к скачанному JSON-ключу, например:

```env
GOOGLE_SERVICE_ACCOUNT_KEY=./secrets/google-sa-key.json
SPREADSHEET_ID=ид_таблицы_из_ссылки
```

`SPREADSHEET_ID` — это длинная строка из ссылки на таблицу:  
`https://docs.google.com/spreadsheets/d/ЗДЕСЬ_ID/edit` — скопируй «ЗДЕСЬ_ID».

---

## Краткий чеклист

1. [ ] Создан проект в Google Cloud.
2. [ ] Включены API: Google Sheets API, Google Docs API.
3. [ ] Создан сервисный аккаунт.
4. [ ] Создан и скачан JSON-ключ, сохранён в `secrets/google-sa-key.json`.
5. [ ] Email сервисного аккаунта добавлен в доступ к таблице с ролью «Редактор».
6. [ ] Если есть Google Документы в настройках — им выдан доступ «Читатель» для того же email.
7. [ ] В `.env` прописаны `GOOGLE_SERVICE_ACCOUNT_KEY` и `SPREADSHEET_ID`.

После этого программа сможет подключаться к таблице и документам по ключу без браузерной авторизации.
