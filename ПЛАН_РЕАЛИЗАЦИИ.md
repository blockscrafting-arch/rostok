## Как заказчице привязать синхробот Дзена (код доступа)

Отдельного «токена» для @zen_sync_bot нет — используется **код доступа**, который выдаёт Студия Дзена.

1. Открыть [t.me/zen_sync_bot](https://t.me/zen_sync_bot) → **Запустить** или `/start`.
2. Открыть [Студию Дзена](https://dzen.ru/media/zen/login) → **Настройки** → **Кросспостинг** → **Telegram**.
3. Нажать **«Получить код доступа»** → скопировать длинную комбинацию букв и цифр (код действует 30 минут).
4. Вставить код в диалог с ботом @zen_sync_bot и отправить.
5. Ввести в боте команду **`/sync`**.
6. В Telegram: канал → **Управление каналом** → **Администраторы** → **Добавить** → найти «Синхробот Дзена» (zen_sync_bot).
7. Отправить боту ссылку на канал (например, https://t.me/your_channel).
8. Выбрать способ публикации: автоматически или вручную.

Канал в Telegram должен быть **публичным**. Подробнее: [dzen.ru/help/ru/channel/cross-platform.html](https://dzen.ru/help/ru/channel/cross-platform.html).

### Этап 2. Google Sheets API (2 дня)

**Файлы:** `src/sheets/*`, `src/scheduler/scheduler.ts`

**Задачи:**

1. **`src/sheets/client.ts`** — инициализация Google Sheets API:
   ```typescript
   import { google } from 'googleapis';
   
   const auth = new google.auth.GoogleAuth({
     keyFile: process.env.GOOGLE_SERVICE_ACCOUNT_KEY,
     scopes: ['https://www.googleapis.com/auth/spreadsheets'],
   });
   
   export const sheets = google.sheets({ version: 'v4', auth });
   ```

2. **`src/sheets/tasks.ts`** — чтение листа «Задания»:
   - Получить все строки с данными.
   - Считывать колонку **«Комментарий»** в поле задачи `task.comment: string | null` (для повторной генерации при «На доработку»).
   - Фильтровать по статусу: «Новое» (нужен сбор семантики), «Согласовано» (нужна генерация), «На доработку» (повторная генерация с учётом комментария), «Одобрено» (нужна публикация).
   - Возвращать массив типизированных объектов `Task`.

3. **`src/sheets/settings.ts`** — чтение листа «Настройки»:
   - Считать поле «Роль» (по умолчанию: «Ведущий агроном питомника с 20-летним стажем»).
   - Считать промпты 1/2/3.
   - Считать ссылки на ДНК бренда и справочник каталога (и загрузить содержимое Google Docs по ссылкам).
   - Считать шаблон UTM, расписание, Channel ID, режим модерации.

4. **`src/sheets/writer.ts`** — запись результатов:
   - Обновить статус строки.
   - Записать заголовки, текст превью, ссылку на картинку, UTM-ссылку, ссылку на пост, источники (граундинг).
   - Записать стоимость (текст/картинка/итого).
   - При повторной генерации после «На доработку»: перезаписывать превью, источники, ссылку на картинку, стоимость по результатам новой генерации; после успеха ставить статус **«Готово к проверке»**. Колонку «Комментарий» не очищать автоматически (по желанию заказчица очищает вручную).

5. **`src/sheets/statistics.ts`** — лист «Статистика»:
   - Добавить новую строку с данными о токенах и стоимости.

6. **`src/scheduler/scheduler.ts`** — главный polling-цикл:
   ```typescript
   async function mainLoop() {
     while (true) {
       const settings = await readSettings();
       const tasks = await readTasks();
       
       // Обработка задач по статусу
       for (const task of tasks.filter(t => t.status === 'Новое')) {
         await semanticsPipeline(task, settings);
       }
       for (const task of tasks.filter(t => t.status === 'Согласовано')) {
         await generationPipeline(task, settings, { isRevision: false });
       }
       for (const task of tasks.filter(t => t.status === 'На доработку')) {
         await generationPipeline(task, settings, { isRevision: true, editorComment: task.comment ?? undefined });
       }
       for (const task of tasks.filter(t => t.status === 'Одобрено')) {
         await publishingPipeline(task, settings);
       }
       
       await sleep(settings.pollInterval);
     }
   }
   ```

**Результат:** Программа читает задания из таблицы, записывает результаты, работает в цикле.

### Этап 4. OpenRouter — генерация текста с граундингом (2,5 дня)

**Файлы:** `src/ai/*` (кроме `image.ts`), `src/utils/utm.ts`

**Задачи:**

1. **`src/ai/client.ts`** — OpenRouter клиент (один для всех моделей):
   ```typescript
   import OpenAI from 'openai';
   
   export const openrouter = new OpenAI({
     apiKey: process.env.OPENROUTER_API_KEY,
     baseURL: 'https://openrouter.ai/api/v1',
   });
   ```

2. **`src/ai/grounding.ts`** — веб-граундинг (фактчекинг):
   ```typescript
   export async function groundArticleFacts(headline: string, keywords: string[]) {
     const response = await openrouter.chat.completions.create({
       model: process.env.OPENROUTER_GROUNDING_MODEL, // perplexity/sonar
       messages: [{
         role: 'user',
         content: `Собери проверенные факты для статьи о растениях.
           Заголовок: "${headline}"
           Ключевые слова: ${keywords.join(', ')}
           
           Верни: зимостойкость (зона USDA), уход (почва, полив, свет),
           сорта, региональные особенности. Только проверенные данные.`,
       }],
     });
     
     const facts = response.choices[0].message.content;
     const citations = response.choices[0].message.annotations
       ?.filter(a => a.type === 'url_citation')
       ?.map(a => a.url_citation.url) ?? [];
     
     return { facts, citations };
   }
   ```
   - Perplexity Sonar автоматически ищет в интернете и возвращает факты с источниками.
   - OpenRouter стандартизирует цитаты в поле `annotations` (формат `url_citation`).
   - Источники сохраняются в колонку «Источники» в Google Таблице.
   - Трекинг токенов для калькулятора.

3. **`src/ai/headlines.ts`** — генерация заголовков:
   - Входные данные: список НЧ-запросов + Промпт 1.
   - Модель: DeepSeek (граундинг не нужен для заголовков).
   - Парсинг ответа: извлечь 30 заголовков.
   - Трекинг токенов.

4. **`src/ai/draft.ts`** — SEO-черновик:
   - Входные данные: заголовок + ключевые запросы + Промпт 2 + **Роль** + **факты из граундинга** + опционально **editorComment**.
   - Необязательный параметр `editorComment: string | null`: если передан (повторная генерация по «На доработку»), в конец user-сообщения добавить блок: «Замечание редактора (обязательно учти при генерации): {editorComment}».
   - Модель: DeepSeek.
   - System prompt: `Ты — {role}. ...` (роль из настроек).
   - В промпт передаётся блок «Проверенные факты из веб-источников: ...» — DeepSeek строит статью на их основе.
   - Генерация **до 4 000 символов**.
   - Трекинг токенов.

5. **`src/ai/humanize.ts`** — очеловечивание:
   - Входные данные: черновик + Промпт 3 + ДНК бренда + ЦА + опционально **editorComment**.
   - Необязательный параметр `editorComment: string | null`: если передан, в конец user-сообщения добавить блок: «Замечание редактора (обязательно учти при генерации): {editorComment}».
   - Модель: DeepSeek.
   - Результат: финальный текст в стиле бренда, **строго до 4 000 символов**.
   - UTM-ссылка в текст НЕ вставляется (кросспост теряет гиперссылки).
   - Трекинг токенов.

6. **`src/utils/utm.ts`** — генерация UTM-меток:
   - Программа берёт тему статьи → ищет в справочнике каталога → собирает UTM по шаблону.
   - **Сохраняет ссылку в колонку «UTM-ссылка» в Google Таблице**.
   - Помощник при оформлении статьи в Дзене копирует ссылку и вставляет как гиперссылку.

7. **`src/ai/cost.ts`** — калькулятор стоимости:
   - Агрегирует `usage` из всех вызовов: граундинг (Sonar) + черновик (DeepSeek) + очеловечивание (DeepSeek).
   - Конвертация в рубли.

**Пайплайн генерации одной статьи:** `src/pipeline/generation.ts`
- Сигнатура: `generationPipeline(task, settings, options?: { isRevision?: boolean; editorComment?: string })`.
- При вызове с `isRevision: true` и `editorComment` передавать `editorComment` в вызовы `draft(...)` и `humanize(...)`.

```
Заголовок + ключевые слова
    ↓
[Perplexity Sonar] → факты + источники
    ↓
[DeepSeek] Промпт 2 + факты (+ editorComment при доработке) → SEO-черновик
    ↓
[DeepSeek] Промпт 3 + ДНК (+ editorComment при доработке) → финальный текст
    ↓
UTM-ссылка → таблица
```

**Результат:** По заголовку система собирает факты из интернета, генерирует статью на их основе, формирует UTM-ссылку и считает стоимость. При статусе «На доработку» то же самое с учётом комментария из таблицы.

### Этап 5. OpenRouter — генерация картинок + S3 (1 день)

**Модель:** `google/gemini-3.1-flash-image-preview` (Gemini 3.1 Flash Image Preview). Поддерживает текст-в-картинку, несколько соотношений сторон (1:1, 9:16, 16:9 и др.), разрешения до 4K. На OpenRouter вызывается через chat/completions с `modalities: ["image", "text"]`. Ориентир по цене: ~$0.03–0.07 за изображение.

**Файлы:** `src/ai/image.ts`, `src/storage/s3.ts`

**Задачи:**

1. **`src/ai/image.ts`** — генерация изображения:
   ```typescript
   const response = await openrouter.images.generate({
     model: process.env.OPENROUTER_IMAGE_MODEL,
     prompt: `Photorealistic photo of ${plantName}, natural lighting, ...`,
     n: 1,
     size: '1024x1024',
   });
   // response.data[0].url или response.data[0].b64_json
   ```
   - Промпт для картинки формируется из названия растения + стиль «фото на телефон».
   - Трекинг стоимости (OpenRouter возвращает цену в ответе).

2. **`src/storage/s3.ts`** — загрузка в Beget S3:
   ```typescript
   import { S3Client, PutObjectCommand } from '@aws-sdk/client-s3';
   
   const s3 = new S3Client({
     endpoint: process.env.S3_ENDPOINT,
     region: 'us-east-1',
     credentials: {
       accessKeyId: process.env.S3_ACCESS_KEY,
       secretAccessKey: process.env.S3_SECRET_KEY,
     },
   });
   
   export async function uploadImage(buffer: Buffer, key: string): Promise<string> {
     await s3.send(new PutObjectCommand({
       Bucket: process.env.S3_BUCKET,
       Key: key,
       Body: buffer,
       ContentType: 'image/png',
       ACL: 'public-read',
     }));
     return `${process.env.S3_ENDPOINT}/${process.env.S3_BUCKET}/${key}`;
   }
   ```

3. Скачать картинку по URL из OpenRouter → загрузить в S3 → получить публичную ссылку → записать в Google Таблицу.

**Результат:** Система генерирует картинку и сохраняет её в S3, ссылка появляется в таблице.

**Задачи:**

## Отказоустойчивость (retry-политика)

**Файл:** `src/utils/retry.ts`

Все вызовы внешних API (OpenRouter, Wordstat, Telegram, S3) оборачиваются в единую retry-обёртку:

```typescript
export async function withRetry<T>(
  fn: () => Promise<T>,
  label: string,
  maxAttempts = Number(process.env.RETRY_ATTEMPTS) || 3,
  baseDelay = Number(process.env.RETRY_BASE_DELAY_MS) || 2000,
): Promise<T> {
  for (let attempt = 1; attempt <= maxAttempts; attempt++) {
    try {
      return await fn();
    } catch (error) {
      if (attempt === maxAttempts) {
        await notify(`❌ ${label}: все ${maxAttempts} попытки провалились.\n${error.message}`);
        throw error;
      }
      const delay = baseDelay * Math.pow(2, attempt - 1);
      await sleep(delay + Math.random() * 1000);
    }
  }
  throw new Error('unreachable');
}
```

**Поведение:**
1. При сбое — повтор через 2 сек → 4 сек → 8 сек (+ случайный jitter).
2. После 3 неудач — статус «Ошибка» в таблице + уведомление в Telegram с описанием.
3. После установки статуса «Ошибка» система не выполняет автоматических повторных попыток. **Повторный запуск:** заказчик вручную меняет статус на **«Согласовано»** (если сбой был на этапе генерации: семантика, текст, картинка) или на **«Одобрено»** (если сбой при публикации в Telegram). На следующей итерации планировщика задача попадёт в соответствующую ветку и выполнится заново.
