# Промпт для Gemini: создание Google Таблицы «Контент-Завод»

Скопируй текст ниже и отдай Gemini (или другому ассистенту), чтобы он создал Google Таблицу с нужной структурой. Таблица будет использоваться программой, которая читает и пишет данные по фиксированным именам листов и колонок — поэтому названия и порядок должны совпадать точно.

---

## Текст промпта (скопировать целиком)

```
Создай новую Google Таблицу с именем «Контент-Завод» (или «Content Factory»). В ней должно быть 4 листа с точными названиями и структурой ниже. Имена листов и заголовки колонок должны совпадать буквально — от них зависит работа скрипта.

---

**Лист 1: «Задания»**

Первая строка — заголовки (ровно в таком порядке, одна колонка = один заголовок):

1. Площадка
2. Ключевое слово
3. Лимит частотности
4. Заголовок
5. Ключевые запросы
6. Статус
7. Превью текста
8. Источники (граундинг)
9. Ссылка на картинку
10. UTM-ссылка
11. Ссылка на пост в TG
12. Стоимость текста (₽)
13. Стоимость картинки (₽)
14. Итого (₽)
15. Дата
16. Комментарий

Со второй строки — пустые строки для данных. В колонку «Статус» пользователь будет подставлять одно из значений: Новое, На согласовании, Согласован заголовок, Генерация, Текст готов, ждём картинку, Готово к проверке, Одобрено на публикацию, Опубликовано, Ошибка, На доработку, Перегенерировать картинку. Можно добавить в лист данные проверки (dropdown или подсказку) для колонки «Статус».

Допустимые значения для «Площадка» на будущее: Дзен, VK, Одноклассники (пока достаточно «Дзен»).

---

**Лист 2: «Настройки»**

Две колонки: **A** — название параметра, **B** — значение. Заполни строки строго с такими формулировками в колонке A (без лишних пробелов):

| A (параметр) | B (значение по умолчанию) |
|--------------|---------------------------|
| Роль | Ведущий агроном питомника с 20-летним стажем |
| Промпт 1 | (оставить пустым или краткий пример: «По ключевому слову {keyword} и запросам {keywords} сгенерируй 30 заголовков статей для блога питомника. Нумеруй с новой строки.») |
| Промпт 2 | (оставить пустым или краткий пример для SEO-черновика) |
| Промпт 3 | (оставить пустым или краткий пример для очеловечивания текста) |
| Промпт картинки | (оставить пустым — для генерации картинки без референса; можно использовать {headline}) |
| Промпт картинки с референсом | (оставить пустым — для картинки с референсом из Справочника фото; можно использовать {headline}) |
| ДНК Бренда | (оставить пустым — сюда вставляется ссылка на Google Doc) |
| Справочник каталога | (оставить пустым — ссылка на Google Doc: раздел → URL) |
| Справочник фото | (оставить пустым — ссылка на Google Doc: сорт/раздел → URL картинки) |
| Шаблон UTM | ?utm_source=dzen&utm_medium=article&utm_campaign={campaign} |
| Модель граундинга | (оставить пустым — используется значение из env на сервере) |
| Модель текста | (оставить пустым — из env) |
| Модель картинки | (оставить пустым — из env) |
| Макс. заголовков | 30 |
| Telegram Channel ID | (оставить пустым или @example_channel) |
| Макс. статей в день | 10 |
| Режим модерации | вкл |
| Время сводки | 21:00 |

Первая строка листа может быть заголовком «Параметр» и «Значение», дальше — пары параметр/значение. Важно: в колонке A должны быть именно эти строки (в т.ч. Промпт картинки, Промпт картинки с референсом, Модель граундинга, Модель текста, Модель картинки, Макс. заголовков, Справочник фото).

---

**Лист 3: «Статистика»**

Первая строка — заголовки (8 колонок по порядку):

1. Заголовок
2. Токены вход
3. Токены выход
4. Модель
5. Стоимость текста (₽)
6. Стоимость картинки (₽)
7. Итого (₽)
8. Дата

Со второй строки — пустые строки. Программа будет дописывать сюда строки по каждой сгенерированной статье. Внизу (через несколько пустых строк) можно добавить блок «Сводка»: расходы за день, за неделю, за месяц, средняя цена статьи — формулы на твоё усмотрение.

---

**Лист 4: «Лог»**

Первая строка — заголовки (4 колонки):

1. Время
2. Действие
3. Результат
4. Ошибка

Со второй строки — пустые строки. Сюда программа будет записывать события (время, действие, результат ок/ошибка, текст ошибки).

---

Итог: один файл Google Таблицы, 4 листа с точными именами «Задания», «Настройки», «Статистика», «Лог», заголовки и параметры — как в инструкции выше. После создания дай ссылку на таблицу и напомни: для работы скрипта нужно выдать доступ на редактирование сервисному аккаунту (email из JSON-ключа Google Cloud).
```

---

## Что сделать после создания таблицы

1. **Скопировать ID таблицы** из ссылки: `https://docs.google.com/spreadsheets/d/**ЗДЕСЬ_ID**/edit` — этот ID подставить в `.env` как `SPREADSHEET_ID`.
2. **Выдать доступ** сервисному аккаунту Google Cloud (email из файла `google-sa-key.json`): «Редактор» или «Просмотр и комментирование» + при необходимости «Редактор» для записи в листы.
