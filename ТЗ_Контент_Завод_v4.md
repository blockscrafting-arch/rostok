# Техническое задание: Контент-Завод для Питомника

**Версия:** 4.4 | **Дата:** 05.03.2026

---

## 1. Суть проекта

Автоматизированная система, которая по одному ключевому слову:

1. Собирает НЧ-запросы из Яндекс Wordstat.
2. Генерирует 30 заголовков.
3. По согласованным заголовкам пишет SEO-статью (до 4 000 символов) в три шага: **веб-граундинг** (факты из интернета) → экспертный черновик → очеловечивание в стиле бренда.
4. Формирует UTM-ссылку на каталог питомника → сохраняет в таблицу.
5. Генерирует фотореалистичную иллюстрацию растения.
6. Публикует текст в Telegram-канал → `@zen_sync_bot` автоматически дублирует в Дзен.
7. Помощник вручную вставляет картинки и UTM-ссылку в статью Дзена (1–2 мин/статья).
8. Считает стоимость каждой статьи в рублях.

**Управление:** Google Таблица.
**Публикация:** Telegram Bot API → официальный кросспост в Дзен (без браузерной автоматизации).

## 2. Алгоритм

### Этап 1. Настройка (однократно)

Лист «Настройки» в Google Таблице:

| Параметр | Что это |
|---|---|
| Роль | Персонаж-автор. По умолчанию: «Ведущий агроном питомника с 20-летним стажем» |
| Промпт 1 | Генерация заголовков из ключей |
| Промпт 2 | SEO-черновик статьи |
| Промпт 3 | Очеловечивание + голос бренда |
| ДНК Бренда | Ссылка на Google Doc с описанием ЦА, тональности, ценностей |
| Справочник каталога | Google Doc: название раздела → URL (например: «Розы» → `https://site.ru/roses`) |
| Шаблон UTM | Формат меток, например: `?utm_source=dzen&utm_medium=article&utm_campaign={topic}` |
| Telegram Channel ID | ID канала для публикации |
| Макс. статей в день | По умолчанию: 10 |
| Режим модерации | Вкл/выкл (вкл = ждёт одобрения перед публикацией) |

Промпты можно менять в любой момент — следующая генерация подхватит новые настройки.

### Этап 2. Сбор семантики

Ключевое слово → Wordstat API → НЧ-запросы → ИИ (Промпт 1) → 30 заголовков → таблица (статус «На согласовании»). **Стоп, ждём заказчика.**

### Этап 3. Генерация текста (с фактчекингом)

Согласованный заголовок проходит три шага:

1. **Веб-граундинг** (Perplexity Sonar): ИИ с встроенным поиском по интернету собирает проверенные факты о растении — зимостойкость, уход, почва, сорта. Результат: фактологическая база со ссылками на источники.
2. **Черновик** (Промпт 2 + факты из шага 1 + Роль): ИИ (DeepSeek) пишет SEO-текст от лица заданного персонажа, опираясь на проверенные данные.
3. **Очеловечивание** (Промпт 3 + ДНК бренда): ИИ (DeepSeek) переписывает черновик — тон бренда, «фишки» питомника, вовлечение.

**Почему Perplexity Sonar:** модель со встроенным поиском по интернету. Каждый ответ основан на реальных веб-источниках → минимум галлюцинаций. Доступна через тот же OpenRouter (один API-ключ, один SDK). Источники логируются для контроля качества.

**Лимит:** до 4 000 символов (одно сообщение Telegram, одна публикация в Дзене).

### Этап 4. UTM-ссылки — как это работает

**Почему ссылка НЕ вставляется в текст Telegram:**
Кросспост `@zen_sync_bot` не переносит форматирование — гиперссылки из Telegram становятся простым текстом в Дзене, raw URL тоже не кликабелен. Поэтому UTM-ссылка вставляется вручную в редакторе Дзена (помощник уже заходит туда для картинок).

**Механизм:**

1. Программа берёт тему статьи (например, «Розы для Сибири»).
2. Ищет в справочнике каталога подходящий раздел. Находит: `Розы → https://site.ru/catalog/roses`.
3. Собирает UTM по шаблону: `https://site.ru/catalog/roses?utm_source=dzen&utm_medium=article&utm_campaign=rozy-dlya-sibiri`.
4. Сохраняет готовую ссылку в колонку «UTM-ссылка» в Google Таблице.
5. Помощник копирует ссылку из таблицы → вставляет как кликабельную гиперссылку в статью Дзена (в тот же заход, когда добавляет картинки).

### Этап 5. Генерация картинки

ИИ (Gemini 3.1 Flash Image Preview, `google/gemini-3.1-flash-image-preview`) генерирует фотореалистичную иллюстрацию растения → сохранение в S3 Beget → ссылка в таблицу.

### Этап 6. Превью (опционально)

Текст + ссылка на картинку → таблица (статус «Готово к проверке»). **Стоп, ждём одобрения.** После обкатки модерацию можно отключить — статьи пойдут в Telegram автоматически.

Если нужны правки — поставить статус **«На доработку»** и заполнить колонку **«Комментарий»** (например: «Сделать акцент на морозостойкости», «Убрать упоминание подкормки в первый год»). Система при следующем обходе запустит повторную генерацию текста (граундинг → черновик → очеловечивание) с учётом этого комментария и снова поставит статус «Готово к проверке».

### Этап 7. Публикация

1. Бот отправляет текст (до 4 096 символов) в Telegram-канал.
2. `@zen_sync_bot` автоматически переносит пост в Дзен как статью.
3. В таблице — статус «Опубликовано» + ссылка на пост.
4. Telegram-бот уведомлений — сообщение заказчику.
5. **Помощник заходит в Дзен** (1–2 мин/статью): ставит обложку, вставляет картинки + UTM-ссылку из таблицы как кликабельную гиперссылку.

**Что переносит кросспост:** только текст (plain text). Форматирование (жирный, курсив, гиперссылки) — теряется. Обложку, картинки внутри текста и UTM-ссылку помощник добавляет в Дзене.

## 4. Архитектура

| Компонент | Технология |

## 6. Отказоустойчивость

При сбое любого внешнего API (OpenRouter, Wordstat, Telegram, S3):

1. **3 попытки** с экспоненциальным интервалом (2 сек → 4 сек → 8 сек).
2. Если все 3 попытки неудачны — статус «Ошибка» в таблице + уведомление в Telegram с описанием проблемы.
3. При восстановлении API — повторный запуск через смену статуса.

**Что делать заказчику:** при статусе «Ошибка» вручную сменить статус в зависимости от этапа сбоя: на **«Согласовано»** — если ошибка была на этапе генерации (семантика, текст, картинка); на **«Одобрено»** — если ошибка была при публикации в Telegram. После смены статуса система при следующем цикле повторит попытку.

## 7. Ограничения и риски

| Риск | Пояснение |
|---|---|
| Лимит 4 096 символов | Статья = одно сообщение Telegram = одна публикация в Дзене. Разбивка на несколько сообщений создаст отдельные посты. |
| Форматирование | Кросспост из Telegram теряет заголовки, жирный, списки. Оформление — вручную в Дзене. |

| Wordstat API | Доступ через Яндекс Директ, получение занимает 1–5 рабочих дней. |
| @zen_sync_bot | Официальный инструмент Яндекса. При изменении его работы — потребуется адаптация. |

### Как привязать @zen_sync_bot

1. Открыть [t.me/zen_sync_bot](https://t.me/zen_sync_bot) → `/start`.
2. В [Студии Дзена](https://dzen.ru/media/zen/login) → **Настройки** → **Кросспостинг** → **Telegram** → **«Получить код доступа»**.
3. Скопировать код (действует 30 мин) → вставить в чат с ботом.
4. В боте ввести `/sync`.
5. Telegram-канал → **Управление** → **Администраторы** → добавить **zen_sync_bot** (расширенные права не нужны).
6. Отправить боту ссылку на канал (например, `https://t.me/your_channel`).
7. Выбрать способ: **автоматически** (рекомендуем).

Канал должен быть **публичным**. К одному Дзен-каналу — один Telegram-канал.

## 10. Критерии приёмки

1. Ключевое слово → Wordstat → заголовки в таблице.
2. Генерация статьи (граундинг + черновик + очеловечивание) → превью в таблице.
3. Источники (веб-ссылки от Perplexity Sonar) — в колонке таблицы.
4. UTM-ссылка на каталог — корректная, в колонке таблицы.
5. Генерация картинки → ссылка в таблице.
6. Публикация в Telegram → кросспост в Дзен.
7. Калькулятор: токены и рубли за статью в «Статистике».
8. Telegram-бот уведомлений работает.
9. Промпты меняются из таблицы — следующая статья использует новый промпт.

**Приёмка:** прогон на 3 статьях от ключевого слова до публикации в Дзене.
